# Session: Đề xuất Ứng dụng Chuyển đổi Markdown Local (Markdown Converter)
Date: 2026-05-17

## Background
- **Mục tiêu:** Xây dựng một ứng dụng nội bộ (Local) nhẹ, đẹp và chuyên nghiệp để chuyển đổi tài liệu Word, DOCX, PDF... sang định dạng Markdown.
- **Yêu cầu cốt lõi:**
  - Giữ nguyên định dạng, bố cục, công thức toán học và biểu đồ để tối ưu cho AI đọc.
  - Sử dụng công nghệ `docling` (từ repo danghoangsqtt-sys/docling) đã tải vào thư mục `docling-env`.
  - Tuân thủ thiết kế và kiến trúc có trong thư mục `DHSYSTEM-TEMPLATE` (files 01, 04, 05).
  - Tất cả dữ liệu sinh ra phải được lưu vào bên trong thư mục hệ thống của ứng dụng, tuyệt đối không lưu vào ổ C:/.
  - Có thể đóng gói dễ dàng để sử dụng trên các máy tính khác.

## Topics Discussed
1. **Kiến trúc Tổng quan:** Chia làm 2 phần Frontend (React/Vite) và Backend (FastAPI Python) để tuân thủ `01-PROJECT_ARCHITECTURE_TEMPLATE.md`.
2. **Quản lý Dữ liệu:** Tạo một thư mục `data/` trong cùng cấp hệ thống dự án, chuyên lưu trữ file tải lên và kết quả Markdown.
3. **Môi trường Python:** Kế thừa thư mục `docling-env` đang có, setup các script chạy backend dựa trên môi trường này để khỏi phải cài đặt lại.
4. **Giao diện & Trải nghiệm Người dùng:** Giao diện tối giản (Minimalist), áp dụng `05-DESIGN_STANDARD.md` với không gian rộng rãi (Luxury/Professional feel). Tích hợp Agentation để tiện dev theo `04-AGENTATION_INTEGRATION.md`.

## Phases
**Phase 1: Foundation & Backend Core**
- Thiết lập thư mục cấu trúc chuẩn.
- Viết API Backend (FastAPI) để nhận file và kết nối với `docling` thực hiện chuyển đổi.
- Cấu hình thư mục lưu trữ `data/`.

**Phase 2: Frontend & UI Integration**
- Xây dựng giao diện React Vite (Drag & drop file).
- Tích hợp Agentation vào môi trường dev.
- Áp dụng các component theo `DESIGN.md`.
- Kết nối API Backend để lấy Markdown trả về và hiển thị/lưu.

**Phase 3: Packaging & Portability**
- Tạo script `.bat` hoặc `.ps1` để khởi động 1-click (khởi động cả Backend API và phục vụ static Frontend).
- Đóng gói ứng dụng để mang sang máy tính khác dễ dàng mà không cần cài đặt lại môi trường rườm rà.

## Decisions
- **Tách biệt Frontend/Backend:** Sẽ dùng React/Vite cho UI và FastAPI (Python) cho Core Logic để dễ tương tác với gói thư viện Python `docling`.
